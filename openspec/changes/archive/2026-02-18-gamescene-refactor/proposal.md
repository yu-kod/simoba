## Why

`GameScene.ts` は 457 行あり、6 つ以上の責務を抱えている: ヒーロー状態管理、敵/リモートプレイヤー追跡、攻撃処理、投射物ライフサイクル、入力/向き制御、ネットワーク連携。今後、構造物 (#20)、ミニオン (#21)、AI (#30) を追加すると手に負えないサイズになる。今のうちに責務ごとのマネージャーに分割し、God Object 化を防ぎつつ各システムを個別にテスト可能にする。

## What Changes

- **EntityManager** を抽出 — 全エンティティの state を保持 (ローカルヒーロー、敵、リモートプレイヤー)。検索・イテレーション API を提供
- **CombatManager** を抽出 — 攻撃ステートマシン、ダメージ適用、投射物ライフサイクル
- **NetworkBridge** を抽出 — GameMode コールバックの配線、アウトバウンドイベント送信
- **GameScene** を薄いオーケストレーターに変更: create → マネージャーに委譲 → update ループで各マネージャーを順番に呼び出し → レンダラー同期
- ゲームプレイの挙動変更なし。純粋な構造リファクタリング

## Non-goals

- 新しいゲームプレイ機能の追加 (構造物、ミニオン、AI)
- ドメインロジック (`src/domain/`) の変更
- ネットワークプロトコルや GameMode インターフェースの変更
- レンダリングレイヤー (HeroRenderer、エフェクト) の変更
- サーバーサイドの変更

## Capabilities

### New Capabilities

_なし_ — リファクタリングのため、新しい仕様レベルの機能は導入しない。

### Modified Capabilities

_なし_ — 要件レベルの変更なし。リファクタリングは `src/scenes/` 内部に閉じており、既存仕様で定義された観測可能な動作は変わらない。

## Impact

- **新規ファイル:** `src/scenes/EntityManager.ts`, `src/scenes/CombatManager.ts`, `src/scenes/NetworkBridge.ts`
- **変更ファイル:** `src/scenes/GameScene.ts` (大幅削減、目標 ~150 行)
- **テスト:** 既存ユニットテストはそのまま通る。抽出したマネージャー用の新規ユニットテストを追加
- **リスク:** 低 — 挙動変更なしの純粋リファクタリング。既存テストスイートで検証可能
