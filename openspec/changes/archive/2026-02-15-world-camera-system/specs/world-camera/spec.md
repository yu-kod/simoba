## ADDED Requirements

### Requirement: ワールド空間の定義

ワールドサイズは 3200x720px でなければならない (SHALL)。ワールド定数は `src/domain/constants.ts` に定義しなければならない (SHALL)。

#### Scenario: ワールドサイズ定数が定義されている
- **WHEN** `src/domain/constants.ts` を参照する
- **THEN** `WORLD_WIDTH = 3200` と `WORLD_HEIGHT = 720` が定義されている

#### Scenario: ワールド空間が Phaser に設定されている
- **WHEN** `GameScene.create()` が実行される
- **THEN** Phaser の物理ワールド境界が 3200x720 に設定されている

### Requirement: ワールド背景の描画

ワールド全体 (3200x720) をカバーする背景が描画されなければならない (SHALL)。画像アセットを使用してはならない (SHALL NOT)。

#### Scenario: 背景がワールド全体を覆う
- **WHEN** GameScene が作成される
- **THEN** 3200x720 の矩形が背景として描画されている

#### Scenario: 背景がジオメトリック描画である
- **WHEN** 背景描画のコードを検査する
- **THEN** Phaser.GameObjects.Graphics を使用しており、画像アセットの読み込みがない

### Requirement: カメラのターゲット追従

カメラはターゲット位置（プレイヤーヒーロー）を追従しなければならない (SHALL)。追従はスムーズな補間（ラープ）を使用しなければならない (SHALL)。

#### Scenario: カメラがヒーロー位置に追従する
- **WHEN** ヒーローが右方向に移動する
- **THEN** カメラの中心がヒーロー位置に向かって移動する

#### Scenario: カメラ追従がスムーズである
- **WHEN** ヒーローが急に方向を変える
- **THEN** カメラは即座にジャンプせず、補間を伴って追従する

### Requirement: カメラのワールド境界制限

カメラはワールド境界 (0, 0, 3200, 720) の外を表示してはならない (SHALL NOT)。

#### Scenario: カメラが左端で停止する
- **WHEN** ヒーローがワールド左端 (x ≈ 0) に位置する
- **THEN** カメラの scrollX が 0 以下にならない

#### Scenario: カメラが右端で停止する
- **WHEN** ヒーローがワールド右端 (x ≈ 3200) に位置する
- **THEN** カメラがワールド右端を超えて表示しない

### Requirement: ヒーロー移動のワールド境界クランプ

ヒーローはワールド境界の外に移動できてはならない (SHALL NOT)。境界制限は domain 層の純粋関数で処理しなければならない (SHALL)。

#### Scenario: ヒーローが左端でクランプされる
- **WHEN** ヒーローが x=0 より左に移動しようとする
- **THEN** ヒーローの x 座標が 0（+ ヒーロー半径）にクランプされる

#### Scenario: ヒーローが右端でクランプされる
- **WHEN** ヒーローが x=3200 より右に移動しようとする
- **THEN** ヒーローの x 座標が 3200（- ヒーロー半径）にクランプされる

#### Scenario: 境界クランプが純粋関数である
- **WHEN** `clampToWorld()` を同じ引数で2回呼び出す
- **THEN** 2回とも同一の結果を返す
